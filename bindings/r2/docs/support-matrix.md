# R2 support matrix

- **Tier:** 0
- **Overall status:** Partial
- **Subpath:** `@redwoodjs/sourdough/r2`
- **Node.js provider:** `@redwoodjs/sourdough/r2/node`

## Reference

- [R2 Workers API](https://developers.cloudflare.com/r2/api/workers/workers-api-reference/)

## Bucket API

| Capability | Status | Notes |
| --- | --- | --- |
| `head(key)` | Supported | Returns object metadata without a body. |
| `get(key, options)` | Partial | Streams full and ranged bodies and applies conditions; SSE-C is not implemented. |
| `put(key, value, options)` | Partial | Supports streams, buffers, strings, blobs, null, metadata, checksums, conditions, and storage-class metadata; SSE-C is not implemented. |
| `delete(key)` | Supported | Deletes one object idempotently. |
| `delete(keys)` | Supported | Deletes multiple objects. |
| `list(options)` | Supported | Supports limit, prefix, opaque cursor, delimiter, `startAfter`, and metadata inclusion. |
| `createMultipartUpload` | Partial | Starts a durable multipart upload with metadata; Cloudflare part-size constraints are not enforced yet. |
| `resumeMultipartUpload` | Partial | Resumes an upload by ID; SSE-C is not implemented. |

## Object and multipart types

| Capability | Status | Notes |
| --- | --- | --- |
| `R2Object` metadata | Supported | Key, version, size, ETags, upload time, range, and storage class. |
| HTTP metadata | Supported | Reads, writes, and applies standard content headers. |
| Custom metadata | Supported | Preserves string metadata. |
| Checksums | Partial | MD5 is generated by default; MD5 and SHA family inputs are validated. Exact multipart checksum and ETag behavior remains incomplete. |
| `R2ObjectBody` readers | Supported | `arrayBuffer`, `bytes`, `text`, `json`, `blob`, body stream, and `bodyUsed`. |
| Multipart part upload | Partial | Uploads numbered parts and validates returned ETags. |
| Multipart complete/abort | Partial | Completes or aborts an upload durably; multipart ETag behavior remains incomplete. |

## Runtime semantics

| Behavior | Status | Notes |
| --- | --- | --- |
| Conditional operations | Partial | Supports ETag and upload-date preconditions from objects or HTTP headers. More edge-case conformance tests are needed. |
| Range reads | Supported | Supports offset, length, suffix, and HTTP Range header forms. |
| Streaming and backpressure | Supported | Provider writes and reads use streams without buffering full objects. |
| Atomic object replacement | Supported | Immutable data files and an atomic metadata-pointer swap prevent partial replacement reads. |
| Bucket isolation | Supported | Each provider instance uses an explicitly configured bucket root. |

## Service contract and providers

The public `R2Bucket` adapter depends on the exported, backend-independent
`R2Service` contract. Third-party providers can implement that contract without
changing application-facing APIs.

| Provider | Runtime | Status | Notes |
| --- | --- | --- | --- |
| `FileSystemR2Service` | Node.js 24 | Partial | Persistent local filesystem provider with safe hashed keys, atomic writes, ranges, conditions, listing, metadata, checksums, and multipart uploads. |
| Nub provider | Nub | Planned | Should implement the same `R2Service` contract and conformance suite. |

## Known gaps

- Server-side encryption with customer-provided keys (SSE-C)
- Exact multipart ETag, checksum, and minimum-part-size behavior
- Full conditional-request edge-case parity
- Storage classes beyond preserving the selected class as metadata
- Conformance testing against Cloudflare-hosted R2

Provider details must not leak into the R2-facing API. Every provider should run
the shared service-contract suite plus the public API compatibility suite.
